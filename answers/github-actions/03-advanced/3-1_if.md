# 解答例：3-1. `if` で step の実行条件を分ける

## 解答

`.github/workflows/3-1-if.yml` を以下の内容で新規に作成します。

```yaml
name: 3-1 if condition

on:
  workflow_dispatch:
    inputs:
      run_step:
        description: Run conditional step
        required: true
        default: false
        type: boolean

jobs:
  job1:
    runs-on: ubuntu-latest
    steps:
      - name: Run job1
        run: echo "job1"

  job2:
    runs-on: ubuntu-latest
    needs: job1
    steps:
      - name: Conditional step
        if: inputs.run_step
        run: echo "conditional step"
```

## 解説

- `workflow_dispatch` に `inputs` を定義すると、Actions 画面から workflow を手動実行するときに値を指定できます。
- この例では、`run_step` という `boolean` 型の input を定義しています。
- `needs: job1` により、`job2` は `job1` の完了後に実行されます。
- `if` に式を書くことで、その step の実行条件を指定できます。
- `if: inputs.run_step` とすることで、`run_step` が `true` のときだけ `conditional step` が実行されます。
- `run_step` が `false` の場合、`job2` は実行されますが、`conditional step` はスキップされます。
