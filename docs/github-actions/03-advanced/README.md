# Step 3: 応用編

このステップでは、GitHub Actions の workflow をもう少し実践的な形にしていきます。

Step 2 では workflow の基本構造を扱いましたが、このステップでは **条件に応じて処理を切り替える方法**、**値を workflow の外に出して扱う方法**、**成果物を保存する方法** を確認します。

このステップでは、次のような要素を扱います。

- **`if` による条件分岐**：step や job の実行条件を定義し、状況に応じて処理を切り替えます
- **イベント・ブランチに応じた実行制御**：起動したイベントやブランチによって、実行する step を変えます
- **variables の利用**：機密情報でない設定値を workflow の外で管理します
- **secrets の利用**：パスワードやトークンのような機密情報を安全に扱います
- **artifact の保存**：workflow 実行中に作成したファイルを保存し、後からダウンロードできるようにします

> 進め方：
>
> 各課題では、その課題で確認したい内容だけを含む workflow を新しいファイルとして作成してください。
