# 3-1. `if` で step の実行条件を分ける

GitHub Actions では、`if` を使って step や job の実行条件を定義できます。

`if` には式を書き、条件が真のときだけその step や job が実行されます。

この課題では、`if` と `needs` を組み合わせて、前の job が完了したあとに、手動実行時に指定した input の値によって step を実行するかどうかを切り替える workflow を作成します。

## プラクティス

次の条件を満たす workflow を `3-1-if.yml` として新規に作成してください。

条件は次のとおりです。

- workflow の起動条件は `workflow_dispatch` にする
- `workflow_dispatch` に input を定義する
- job を 2 つ定義する（`job1`、`job2`）
- `job1` では `echo "job1"` を実行する
- `needs` を使って `job1` → `job2` の順に実行されるようにする
- `job2` に `if` を使った条件付き step を定義する
- 条件付き step は、input の値が `true` のときだけ実行される
- 条件付き step では `echo "conditional step"` を実行する

`workflow_dispatch` の input は、次のように定義します。

```yaml
on:
  workflow_dispatch:
    inputs:
      run_step:
        description: Run conditional step
        required: true
        default: false
        type: boolean
```

> ヒント:
>
> - workflow_dispatch は、GitHub Actions の画面から workflow を手動実行できるようにするイベントです
> - workflow_dispatch に input を定義すると、手動実行時に値を指定できます
> - input の値は inputs.<input名> の形で参照できます

> 必要に応じて、次の公式ドキュメントを参照してください。
>
> - [Expressions](https://docs.github.com/en/actions/concepts/workflows-and-actions/expressions)
> - [Workflow syntax for GitHub Actions - on.workflow_dispatch.inputs](https://docs.github.com/ja/actions/reference/workflows-and-actions/workflow-syntax#onworkflow_dispatchinputs)
> - [Workflow syntax for GitHub Actions - jobs.<job_id>.steps[*].if](https://docs.github.com/ja/actions/reference/workflows-and-actions/workflow-syntax#jobsjob_idstepsif)

## 確認

- 変更を push し、Actions 画面から workflow を手動実行する
- `job1` の完了後に `job2` が実行されることを確認する
- `run_step` を `true` にして実行したとき、`job2` の中で `conditional step` が実行されることを確認する
- `run_step` を `false` にして実行したとき、`job2` の中で `conditional step` がスキップされることを確認する
